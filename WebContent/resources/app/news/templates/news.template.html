Peach Release - Version 3.2 - May 12th, 2016

API's

Enhancements

Two enhancements have been made to the Update Subscription API call:

  • The client order id has been added as a field that can be returned and populated based on information in the call.
  • The Renewal Count has also been added as a field that can be returned.

Customer View

Demo Date and Batch Tracking ID

In our last release, we added the option to key the demo date and batch tracking id on each individual demographic when entering an order. With this code release, you are now able to add/update this information in the demographic section of Customer View. Please contact your Account Manager with any questions.

Omail

Upgraded version of WYSIWYG editor

Our WYSIWYG editor has now been upgraded to the most recent version of CKEditor - CKEditor 4.5.7 (revision e98277f).
New features include an updated interface and the addition of the Styles drop-down which allows you to easily apply customized styles and semantic values to content created in the editor.
The user guide can be found here CKEditor User Guide.

Reports

Omail - Summary by Link Category

The Summary by Link Category report has now been enhanced to allow you to run it over user specified date ranges. Previously you could only select an entire month period. This report is an excellent way to view the success of your categorized links. Please contact the Omail team to review this functionality.

"Let's-a go!"

Luigi Release - Version 3.1 - March 31st, 2016

API's

Enhancement: Allow in 1 call

Within the Save Customer and Order API call, you can now submit both a Paid order and a Comp order in one call. This will now allow you to offer products that are both free and paid and send the information in one call.

olytics

Enhancement: General Traffic report

The General Traffic Report has been modified by adding a "show all" option on the page. By default, we only report on the top 20 rows. You can now click on the "show all" to see the visit counts for each tag, all on 1 page.

Update: Anonymous/Known skittle in OnQ

A new option has been added to the Anonymous/Known skittle in OnQ. You can now select "Known with olytics data". Prior to this release, the "known" option included everyone on your database that was not classified as "anonymous". Your 3 options now for selecting data are (1) Anonymous (2) Known without olytics data (3) Known with olytics data. Please contact your account manager with any questions.

OnQ

New Feature: Renewal Count

You can now keep track of how many times each customer renewed their paid subscription for each product. Every time a renewal is added to a customer's record, the renewal count will be incremented by 1. This new data field allows you to assess a lifetime value for each customer. You will be able to query this information in OnQ and output it. Plus, you will see it on the customer's record in Customer View.

Processor

Enhancement: Rules for digital edition requests

Your account manager can now create rules which will automatically override any prior optouts for your digital editions when your customer now requests the digital edition. Please contact your Account Manager for further details.

Tags: API olytics OnQ Processor

Mario Release - Version 3.0 - February 18th, 2016

API's

A new get API for olytics data

You now have the ability to do lookups based on attribute counts for olytics data as it relates to a behavior for a designated time-frame.

Customer View

Age of a demographic value

You will now be able to see the "Demo Date" and also the "Batch Tracking ID" for each demographic value. This is very useful information for audited magazines. These data fields have always been available in OnQ - they are now visible in each customer's record in Customer View.

Company name only customers

When creating a new customer in Customer View, you are now able to create a customer with company name only. Before, we were requiring an address as well. This is beneficial when you have limited contact data but have additional data such as behaviors or demographics. This allows you to build the customer profile through increased engagement.

Removing an email address from a customer

In those situations where it is imperative that a customer's email address is totally removed from your database, you will be able to do so, since the Deactivate link will always be available. This deactivates the email, removing it from view and any means of output from OnQ.

Sally Release - Version 2.8 - January 7th, 2016

API's

Email lookup

The Email look up API was modified to allow for the new gTLD's (global top level domains), which can be longer than standard four characters.

Promotion code lookup

The Promotion code lookup API has now been updated to return a promo code status code, a requested version, and a calculated bill me, if needed.

Subscription lookup by email address

A new API has been created to enable you to look up subscription information, based on an email address. Here's our comprehensive API Suite documentation.

Customer View

Enhanced memo for Nixies

For clients that require more than one nixie before deactivating the postal delivery address, the Memo field in Customer View will now show when the nixie counter has been incremented.

olytics

olytics data available for OnQ outputs

When selecting olytics data in OnQ, the following fields can now also be in your output file: IP Organization, Anonymous, Device Type, and Operating System.

olytics reporting

Within the Reports tab of the my.omeda.com portal, specialized reporting for olytics is now available within the olytics section. General Traffic, Visitor Technology, and Pathing reports provide a deeper view into your website visitors.

Omail

gTLD

Due to the release of new Global Top Level Domains: gTLD's, the my.omeda.com portal has been enhanced to allow for the use of these new domains such as .accounting or .fitness or .photography and even .beer.

Processor

OEC - Open Ended coding

The "Reassign Values" section of the OEC program has been modified to now allow users to reassign a value of "Other" to previously coded values.

Reports

Add/Kill by Reason report

The Add/Kill report that is generated during an Issue Close has been modified by adding reason codes on the detail section. This is accessible by clicking on a number on the summary.

Promo Code report

The promo code report has been modified by adding the quantity sent and return percentage. Clients can record the quantity sent number within the Campaign tool when setting up new promo codes to enable this new data.

Sales Tax report

The details within the sales tax report have been modified by adding a county break out This is beneficial to clients that are required to use individual county sales tax tables.

"What do you mean Beethoven wasn't so great?!"

Schroeder release - v2.7 - November 20, 2015

olytics

Customer View: Enhanced olytics data

Clicking on View Visits within an olytics Behavior will show 5 new columns of data: Date, Device Type, Browser, Operating System, and Referring Domain. This is an excellent way to see how an individual customer is interacting with your content.

OnQ: Enhanced olytics data

As we are now capturing the IP address from which a customer is accessing your content, by using a 3rd party look-up service, we can assign more data points. You will now see the following fields in OnQ to query this data: IP Organization, IP Location, IP Region, IP Country, IP City, and IP Zip/Postal Code.

Reporting: General Traffic report

This new report allows you to view overall visits and unique customer visits in a time graph by both Known and Anonymous visitors. You can also drill down into specified behaviors, categories and tags.

Reporting: Omail Deployment Delivery report

The Omail Deployment Delivery report will now show the number of clicks on links with olytics tracking enabled. This additional reporting will appear in the Response Categories, Delivery by Domain, and Tracking Links Clicked sections. Please contact the Omail team with any questions regarding this new data.

Reporting: Pathing report

This report allows you to view visits by referring domain. The data is presented on a time graph.

Reporting: Visitor Technology report

This report allows you to view Visits by Browser (Chrome, Firefox, Safari, etc.), Operating System (Windows, Android, iOS, etc.), and Device (personal computer, smartphone, tablet, etc.). The data is represented on a time graph by Known and Anonymous visits.

Omail

Clarification for SpamAssasin scoring

When the Omail application was unable to connect to SpamAssasin for spam scoring, a score of 99 was displayed. This will now appear as Not Available to avoid confusion that a score was too high.

OnQ

Group Active Count

Within OnQ you can now query your paid groups to find the groups with x number of active members. This is helpful when doing your modeling or even determining the different price points that you may want to use for Renewals.
